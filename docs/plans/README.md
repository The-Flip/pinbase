# Plans

This directory contains historical planning documents.

These docs are useful for understanding prior design thinking, tradeoffs, and implementation sequencing, but they are not canonical product or system documentation. A plan doc becomes outdated as soon as implementation lands.

Use the stable docs in `docs/` for the current description of how the system works. Use `docs/plans/` for historical context, rationale, or the state of thinking at a particular moment in time.
